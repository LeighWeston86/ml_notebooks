Machine learning semiconductor band gaps¶

Leigh weston, July 2017¶

The band gap is the fundamental property for a semiconductor, particularly for photovoltaics applications. Quaternary semiconductors with general formula I2-II-IV-VI4 are promising solar absorbers, however there are thousands of possible cation-anion comibations; this is far too many to perform accurate quantum mechanical caluclations on.

I have performed accurate quantum mechanical calculations on a subset (n = 200) of these materials, and used the results to train a machine learning model. The trained model can be used to predict the band gaps of 1568 possible I2-II-IV-VI4 semiconductors.

I tried a number of machine learning aglorithms for regression, including support vector machines, decision trees, and tree-based enemble methods. For determining the magnitude of the fundamental gap, support vector regression with a radial bias kernel performed the best, and this method is used below.

Organising the data¶

The first step is to organize the data and set up the descriptors for fitting.

Each I2-II-IV-VI4 compound has four elements; 3 cations with valence I, II, and IV, and one anion with valence VI. For descriptors, I am using simple, element-specific features. I am using the electronegativity, ionic radius, and row in the periodic table; for the cations, I also use the column in the perodic table. There are 15 descriptors in total (4 for each cation, 3 for the anion).

The input is a list of the cations and and anions in the compound, ['I', 'II', 'IV',, 'VI']; gen_x() returns the 15-dimensional vector of features for the compound.

def gen_x(material_array):
    z = {}
    for material in material_array:
        z.update(eval(material))
    
    return pd.Series(z)

Default parameters¶

At first, I just want to get a feeling for how well the model works with default parameters, so I randomly select 75% of the data to train a support vector regressor with a radial bias kernel; the remaining 25% of the data can be used to verify the accuracy of the predictions.

Support vector machines fit a hyperplane in a multidimensional space; in regression, there is a hyperplane margin as well as a radial bias function. The distance between two points is defined as the Euclidean distance and so it is important to properly scale the features. I use the MinMaxScaler here.

In [45]:

Optimise parameters¶

Using 10-fold cross validation, we achieve a root mean squared error of 0.384 eV. This is OK, but not great. The SVR is using a radial bias kernel, which has parameters $C$ and $\gamma$. The $C$ parameter controls the tradeoff between the "softness" of the plane margin, and the "steepness" of the fitted plane; the $\gamma$ parameter determines how rapidly the radial bias function, $\exp \left[\gamma (x - x')\right]$, decays. These can be optimized to give better predictions.

I can do a grid search over $C$ and $\gamma$.

At first, I do a coarse grid search over a wide range using 10-fold cross validation.

Using grid search, I identified that $C = 127.4$ and $\gamma = 0.127$ are optimum parameters for our model.

With these parameters, I find a root mean squared error of 0.08 eV. This is really an excellent result. This is around what is expected in the numerical noise of the band gaps that were used to train the model. This tells me that the regressor is performing as well as it possible could.
